# Glossário do Domínio (AMS)

1.  *Asset (Ativo):* Qualquer item de hardware (portátil, monitor, servidor) que tenha valor para a organização e necessite de ser rastreado.
2.  *Intake:* O processo inicial de receção e registo de um novo ativo no sistema.
3.  *Serial Number (Número de Série):* Identificador único atribuído pelo fabricante ao hardware, usado para prevenir duplicados.
4.  *SKU (Stock Keeping Unit):* Código identificador do tipo de produto/modelo vendido.
5.  *Validation (Validação):* Conjunto de regras lógicas aplicadas aos dados de entrada para garantir a sua qualidade (ex: formato de data, campos não vazios).
6.  *Duplication Check:* Verificação automática para confirmar se um ativo com o mesmo identificador já existe na base de dados.
7.  *Stakeholder:* Qualquer pessoa ou grupo interessado no sucesso do projeto (ex: Gestor de IT, Auditores).
8.  *Audit Log:* Registo imutável de todas as ações realizadas no sistema (quem criou, o quê e quando).
9.  *Baseline:* O conjunto de funcionalidades padrão definido no caso de estudo do curso (AMS).
10. *Use Case:* Descrição de como um utilizador interage com o sistema para atingir um objetivo específico.
11. *Acceptance Criteria:* Condições que um requisito deve satisfazer para ser aceite pelo cliente.
12. *API Endpoint:* Ponto de acesso onde o sistema recebe pedidos de criação ou consulta de ativos.
13. *Fail-Fast:* Princípio onde o sistema deve rejeitar dados inválidos imediatamente, retornando um erro claro.
